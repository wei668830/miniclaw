📖 项目概述

MiniClaw（小搔）是一个面向 AI 智能体的编排与工具管理框架。它提供了一个交互式命令行界面和 REST API 服务，让大语言模型（LLM）能够自主调用各类工具完成复杂任务。

借鉴 CoPaw 项目的设计理念，MiniClaw 将 LLM 的对话能力与文件操作、Shell 命令、浏览器自动化、屏幕截图等能力有机结合，支持子任务拆分、记忆压缩、自主推进等高级特性。

⚙️ 核心功能详解

🤖 智能体编排

基于 BaseLLMClient 抽象基类，支持非流式(chat)和流式(stream)两种对话模式。统一返回结构 LLMResponse/LLMStreamChunk，包含内容、Token 消耗和费用信息。

🔧 工具系统

LLMToolsManager 管理所有注册工具，支持按名称启用/禁用（LLM_ENABLED_TOOLS/LLM_DISABLED_TOOLS）。自动检测 LLM 常见的"多 JSON 对象拼接"错误并给出友好提示。

🧩 子任务执行器

Clerk（办事员）使用独立对话上下文执行子任务，支持递归调用（带层级追踪防止无限循环）。执行完毕后自动总结任务结果。

📋 任务规划者

Planner 根据需求生成结构化 Markdown 计划，包含任务说明、进度跟踪和明细执行步骤，自动保存至 ~/.miniclaw/plans/。

🧠 记忆管理

Memory 使用 LLM 自动精简对话历史，保留关键信息，去掉冗余的工具调用细节。适用于超长任务的上下文压缩以节省 Token 费用。

🚀 自主推进模式

支持代理模式（agent mode），LLM 自行判断何时继续推进任务。通过 turn_taking.py 中的决策者提示词，实现无限推进直到任务完成。

🌐 Web 服务

基于 FastAPI 提供 REST API 和 Server-Sent Events 流式接口，支持 CORS 跨域。聊天上下文按 CID 持久化存储。

📝 日志系统

基于 loguru 的日志系统，支持控制台和文件双输出，文件按日轮转，保留 14 天并自动压缩。

✨ 总结与特点

  • 🎯
    自主任务执行

    支持代理模式（Agent Mode），LLM 可自主判断何时继续推进任务，实现端到端的自动化执行流程。

  • 🧩
    模块化工具系统

    统一的工具注册/执行机制，支持按需启用/禁用。自动检测 LLM 的工具调用错误并提供友好提示。

  • 🔗
    子任务编排

    办事员（Clerk）可递归执行子任务，带层级追踪防止无限循环。配合 Planner 可生成结构化执行计划。

  • 💾
    智能记忆管理

    使用 LLM 自动精简对话历史，保留关键上下文的同时大幅减少 Token 消耗。适用于超长任务的上下文压缩。

  • 🌐
    双模运行

    同时提供交互式 CLI（基于 Rich + prompt_toolkit）和 Web API（基于 FastAPI + SSE）两种交互方式。

  • 🔌
    多模型支持

    基于 LiteLLM 统一接口，支持 OpenAI、DeepSeek、Claude、Gemini 等主流模型，也支持私有化部署的兼容 OpenAI 协议的服务。

  • 🛡️
    安全防护

    Shell 命令含自保护机制（防止误杀 Python 进程）、浏览器空闲自动回收、临时脚本定期清理。

  • 📊
    成本可视化

    每次 LLM 调用均记录 Token 消耗和费用（USD），帮助用户了解和控制使用成本。
