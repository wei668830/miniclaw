Overview

项目概述

MiniClaw 是一个基于大语言模型（LLM）驱动的智能体自动化框架，借鉴了 CoPaw 项目的设计理念。 它允许用户通过自然语言与 AI 交互，自动完成从简单的文件读写到复杂的多步骤任务编排。

🎯

核心定位

AI 脚本手架工具 —— 通过自然语言驱动 AI 智能体完成各类自动化任务，是开发者的智能助手。

⚡

运行方式

  • CLI 交互模式 — 命令行对话界面，支持 Rich 实时 Markdown 渲染
  • Web 服务器模式 — FastAPI + Streaming SSE，支持 Web UI 前端调用
🧠

AI 驱动

基于 LiteLLM 抽象层，支持 DeepSeek、OpenAI、Claude 等数十种大模型。通过工具调用（Function Calling）实现自主操作。

🔧

适用场景

  • 文件读写与代码生成
  • 浏览器自动化操作
  • 多步骤任务规划与执行
  • 后端服务启动管理
  • 智能对话与记忆管理
📌 设计理念

"小搔"（MiniClaw 的中文名）的核心理念是让 AI 自主完成任务： 只需要告诉 AI 你想要什么，它就能自动拆解任务、调用工具、推进执行，并在需要时向你汇报结果或请求进一步指示。

Architecture

系统架构

MiniClaw 采用分层架构设计，各模块职责分明，耦合度低，易于扩展。

👤 用户输入
→
🎯 CLI / FastAPI
→
🧠 LLM Agent
→
🔧 工具执行
→
✅ 结果反馈
🎨

交互层

  • CLI — Rich 终端交互，支持 Markdown、思维链、工具调用实时渲染
  • FastAPI Web — REST API + SSE 流式响应，支持前端页面
  • 命令系统 — /help, /chat, /agent, /clear, /memory, /quit 等
🧩

Agent 层

  • BaseLLMClient — 抽象接口，定义 chat() 和 stream()
  • LiteLLMClient — 默认实现，支持数十种大模型
  • LLMToolsManager — 工具注册、启用/禁用、调度执行
  • LLMConfigurator — 模型/API Key/Base URL 配置管理
⚙️

执行层

提供 15+ 个内置工具，涵盖文件操作、系统命令、浏览器自动化、截图、进程管理等完整能力。

  • 文件工具：read / write / edit / append / tail / search / find
  • 系统工具：shell / launch_detached / time
  • 浏览器工具：playwright 全面控制
  • 视觉工具：desktop_screenshot / view_image
  • 特殊工具：subtask_executor / make_plans
🔄 对话流处理流程

用户输入 → LLM 流式响应（支持思维链 reasoning_content）→ 若触发 tool_calls → 解析工具名和参数 → 执行工具（文件/Shell/浏览器等）→ 工具结果追加到消息历史 → 继续下一轮 LLM 调用（自动循环）→ 非工具调用结束时，Agent 模式下自动判断是否继续推进，Chat 模式下等待用户输入。

Features

核心功能特性

MiniClaw 提供丰富而强大的功能，覆盖开发与运维的多个方面。

💬

智能对话交互

  • 支持流式 Markdown 实时渲染（Rich Live 组件）
  • 支持 DeepSeek R1 等模型的思维链（reasoning_content）展示
  • 双模式运行：Agent 模式（自动推进）和 Chat 模式（一问一答）
  • Token 用量和费用实时统计
📂

完整文件操作

  • 读取：支持指定行范围，自动检测编码
  • 写入：自动为 CSV/TXT 添加 UTF-8 BOM 兼容 Excel
  • 编辑：精确查找替换，保留原格式
  • 追加：安全追加内容到文件尾部
  • 尾部读取：高效读取日志文件的最后 N 行
🔍

文件搜索系统

  • grep_search — 基于 ripgrep 的内容搜索，支持正则、上下文行、文件名过滤
  • glob_search — 基于 glob 模式的文件查找，支持通配符和目录限定
🌐

浏览器自动化

基于 Playwright 的全功能浏览器控制，支持 25+ 操作：

  • 导航：打开/跳转/后退/等待
  • 交互：点击/输入/拖拽/悬停/选择
  • 数据：截图/PDF/控制台日志/网络请求
  • 管理：Cookie/标签页/对话框/CDP远程调试
🖥️

桌面截图 & 图像识别

  • 跨平台桌面全屏截屏（Windows/macOS/Linux）
  • macOS 支持截取指定窗口
  • AI 可直接"查看"图片文件内容
⚡

后台进程管理

启动独立进程脱离 Python 程序独立运行：

  • Windows：通过 .bat + start /min 实现
  • Linux/macOS：通过 nohup 实现
  • 适用场景：Spring Boot、Node.js、yarn/npm 构建等
  • 所有输出重定向到指定日志文件
📋

子任务执行器（Clerk）

  • 独立的对话上下文执行子任务
  • 执行完毕后返回精简报告
  • 支持层级嵌套（但限制不允许嵌套调用以避免无限循环）
  • 自动总结执行结果（成功/失败 + 执行情况）
📐

任务规划器（Planner）

  • 自动将复杂任务拆分为结构化执行步骤
  • Markdown 格式计划文档，自动保存到本地
  • 每个步骤明确工具、校验方法、依赖关系
  • 支持进度跟踪（[X]/[ ] 标记）
🧠

记忆管理系统

  • 自动压缩超长对话上下文，保留关键信息
  • 记忆缓存以 Markdown 文件形式持久化到磁盘
  • 支持查看/检索历史记忆文件
  • 自动清理超过 1000 条的旧记忆
🔌

对话轮次管理

  • Agent 模式下自动判断是否继续推进任务
  • 基于最近 N 条消息做决策（避免上下文太长）
  • 可自定义决策提示词（CHAT_ADVANCE_SYSTEM_PROMPT）
  • 避免循环对话流和无明确指向的继续处理

Modules

核心模块详解

深入了解 MiniClaw 各核心模块的设计与职责。

🏗️

__main__.py — 双模式入口

支持两种运行模式：

  • miniclaw — 默认进入 CLI 交互模式
  • miniclaw server — 启动 FastAPI Web 服务
  • miniclaw cli — 显式启动 CLI 模式
  • miniclaw -v — 查看版本号

通过 argparse 解析命令参数，支持 --host/--port/--log-level 等配置。

🧬

base_llm_client.py — LLM 抽象层

定义了统一的 AI 交互接口：

  • LLMStreamChunk — 流式响应块，包含 delta、finish、tool_calls、token 统计
  • LLMResponse — 非流式完整响应，包含 content、reasoning_content、cost
  • ToolResponse — 工具执行结果，支持文本和图片混合内容
  • BaseLLMClient — 抽象基类，定义 chat() 和 stream() 接口
🔧

llm_tools_manager.py — 工具调度中心

  • 15+ 工具的注册、启用/禁用管理
  • 通过 LLM_ENABLED_TOOLS 和 LLM_DISABLED_TOOLS 环境变量控制
  • 智能检测 LLM 拼接 JSON 对象错误，返回友好提示
  • 工具定义完整（名称、描述、参数 JSON Schema），供 LLM 调用
📞

cli/main.py — CLI 交互引擎

  • 完整的命令系统（/help, /chat, /agent, /clear, /memory 等）
  • 流式 Markdown 渲染优化（缓冲刷新减少终端闪烁）
  • Runtime Mode 切换：Agent（自动推进） / Chat（一问一答）
  • 记忆缓存上下文注入
  • Token 和费用统计展示
🧑‍💼

cli/clerk.py — 子任务执行器

  • 独立的 Clerk 类，拥有独立的对话上下文
  • 排除 subtask_executor 工具防止递归嵌套
  • 执行完毕后自动调用 LLM 总结执行结果
  • 层级追踪（layer），便于日志调试
  • 支持自动推进（类似 Agent 模式）
📐

cli/planner.py — 任务规划器

  • 继承 Actor 基类，接收用户需求自动生成计划文档
  • 计划文件自动保存到 ~/.miniclaw/plans/ 目录
  • 包含 > 10 条详细规划规则（步骤粒度、校验方法、依赖关系）
  • 生成失败时自动分析原因并返回
Security

安全与防护机制

MiniClaw 内置了多重安全防护措施，防止 AI 误操作导致系统问题。

🛡️

自杀式操作防护

系统提示中明确告知 LLM 不要通过 Shell 工具停止当前进程（PID 已注入提示词）。Git 历史显示专门提交了"安全检查：禁止执行可能导致自杀式操作的命令"。

🔗

JSON 拼接检测

工具管理器内置正则检测，当 LLM 错误地将多个 JSON 对象拼接发送时，自动拦截并返回清晰错误提示，避免误操作。

🔑

敏感信息掩码

security.py 提供密码掩码功能，在终端展示 API Key 时仅显示前3位和后3位，中间用星号代替。

🚫

工具权限控制

通过 LLM_ENABLED_TOOLS 和 LLM_DISABLED_TOOLS 环境变量，可精确控制 AI 可使用和禁用的工具列表。

🔄

递归嵌套防护

Clerk（子任务执行器）的 tools 列表中排除了 subtask_executor，从机制上防止了 Clerk 无限递归调用。

⏱️

超时控制

Shell 命令执行支持超时设置（默认 60 秒），防止长时间运行的命令阻塞系统。

Quick Start

快速开始

5 步即可启动 MiniClaw，体验 AI 驱动的自动化工作流。

1️⃣

安装 Python 环境

需要 Python 3.11+。推荐使用 Miniconda 管理环境：

conda create -n miniclaw311 python=3.11
conda activate miniclaw311
2️⃣

安装 Poetry 和依赖

pip install poetry
cd MiniClaw/
poetry install
3️⃣

配置环境变量

复制 .env.example 为 .env，填入你的 API Key 和模型配置：

LLM_MODEL=deepseek/deepseek-v4-flash
LLM_API_KEY=sk-your-key-here
LLM_BASE_URL=https://api.deepseek.com
4️⃣

安装额外依赖（可选）

pip install playwright
playwright install
pip install uvicorn
5️⃣

启动 MiniClaw

修改 miniclaw.bat 中的 Python 路径后直接运行：

miniclaw.bat

或直接使用 Python 命令启动：

python -m miniclaw

启动后你将看到炫彩的终端界面，输入 /help 查看所有可用命令。

API

Web API 接口

FastAPI 服务提供标准 REST 和 SSE 流式接口，方便构建前端应用。

接口 方法 说明 请求体
/ GET 根路径，返回项目描述信息 —
/api/version GET 返回当前 MiniClaw 版本号 —
/api/llm/chat POST 非流式对话（单轮），自动管理聊天上下文 {cid, messages, model?, temperature?, max_tokens?}
/api/llm/stream POST SSE 流式对话，支持工具调用自动化 {cid, messages, model?, temperature?, max_tokens?}
📡 SSE 事件流

/api/llm/stream 使用 Server-Sent Events（SSE）协议，包含以下事件类型：
event: chunk — 文本增量内容
event: tool_call — 工具调用（含完整工具参数）
event: done — 对话完成（含 Token 统计和费用）
event: error — 错误信息

项目根目录提供了两个前端演示页面：chat.html（普通）和 chat_stream.html（流式），可直接在浏览器中打开使用。

Summary

项目特点总结

MiniClaw 的核心竞争力和独特价值。

🎯

极简交互

自然语言驱动，无需学习复杂命令。告诉 AI 你想做什么，剩下的交给它。

🧩

高度可扩展

基于抽象接口设计，Provider、Tool、Agent 均可自由替换和扩展。新增一个工具只需注册即可。

🔄

智能任务推进

Agent 模式可自动判断任务是否完成，自主推动对话继续执行，无需人工干预。

💰

成本可控

实时显示 Token 用量和费用，记忆压缩机制减少长对话开销。支持配置不同模型处理不同类型任务。

🛡️

安全优先

多重安全防护：自杀式操作防护、JSON 错误检测、工具权限控制、递归嵌套防护。

📦

开箱即用

Poetry 管理依赖，一个 .env 文件完成配置，Windows/Linux/macOS 全平台支持。
